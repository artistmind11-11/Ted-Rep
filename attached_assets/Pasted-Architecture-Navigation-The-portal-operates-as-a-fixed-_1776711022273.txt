Architecture & Navigation
The portal operates as a fixed-position overlay (position: fixed; inset: 0) that locks the underlying site scroll (overflow: hidden on body) and renders a self-contained application shell. It never navigates — all 9 sections are pre-rendered in the DOM as display: none panels; tab clicks toggle visibility via classList. Switching panels always resets scrollTop to zero.
Session behavior: Charts initialize once on first portal entry (chartsInit flag), destroying and re-rendering only on dark/light mode toggle to recolor canvas elements. No data persistence — all state is in-memory.

3. Tasks
Purpose: Kanban-style task management across all active engagement workstreams.
KPI Strip: Total (24), In Progress (8), Completed (11), Overdue (0).
Kanban Board (3 columns):
To Do: Standard backlog. Cards without left-border accent. Badges: b-wait (gold, Standard) or b-pend (gold, Review).
In Progress: Active tasks. Priority items carry a 2px solid #c8b97a left-border — the Ledger "premium status" indicator from the design system. Badges: b-pend (Urgent) or b-prog (blue-grey, Active).
Completed: Historical record. Cards rendered at 50% opacity — present but visually receded. All badges b-done (green).
Each card: task title (Inter 300), project category label (muted, 10px), bottom row with due date + status badge.
Workflow: No drag-and-drop in this implementation — cards are static. Column counts in the header update to reflect data. Clicking a card has hover feedback (background lift) but no detail panel.

4. Approvals
Purpose: Decision queue for legal, financial, governance, and document items requiring principal authorization.
Approval Item Structure:

Type label (76px fixed, uppercase, muted) — Legal / Financial / Document / Governance
Title + subtitle describing the item and its origin/context
Two action buttons: Approve (a-ok) and Reject (a-rej)

Interaction: Clicking either button sets the row to 38% opacity, disables pointer events, and replaces the button pair with a status badge (b-done or b-due). This is destructive — no undo in-session.
Recently Actioned Table:

Columns: Item, Type, Date, Action badge
Shows last 3–4 actioned items with final status
Green badge = approved, red badge = rejected

Workflow: Items flow in from external submission (Legal Counsel, auto-generated invoices, TED-prepared documents). Principal reviews and acts. Approved items trigger downstream notifications (simulated in copy).

Cross-Panel Design Patterns
KPI cards: All panels open with a 4-column KPI strip. Top edge gets a gold gradient hairline on hover (::before pseudo-element, background: linear-gradient(90deg, transparent, gold, transparent)). Progress bars are 1px height, 4% white base, gold fill.
Table rows: Tonal hover only (1% white lift) — no border highlight. All amounts in Manrope 300 for numeric readability. Badge color system is consistent across all panels: green = positive/done, gold = pending/active, blue-grey = neutral/in-progress, orange = high-priority, red = error/rejected.
Portal-to-site relationship: Exit button restores body.overflow and hides the portal. Dark mode toggle on the site side destroys and rebuilds all Chart.js instances to match the new canvas colors — this is the only scenario where charts re-initialize.
